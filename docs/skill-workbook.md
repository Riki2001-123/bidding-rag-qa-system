# 技能学习产出模板

这份模板现在同时服务两种节奏：

- `docs/study-plan.md` 的 2 周项目接手计划
- `docs/skill-roadmap-4weeks.md` 的 4 周扩展路线

推荐做法：

1. 每天结束前先填“每日学习台账”。
2. 每周结束后再填“复盘模板”。
3. 做小改动时补“练习变更记录”。
4. 最后用“10 分钟讲解稿模板”做毕业演示。

## 2. 系统主链路图模板

把你自己的理解写成图，不要只抄文档。

```mermaid
flowchart LR
    A["原始资料"] --> B["字段整理 / Excel"]
    B --> C["导入接口"]
    C --> D["数据校验与入库"]
    D --> E["文本切块"]
    E --> F["向量化"]
    F --> G["结构化检索 + 语义检索"]
    G --> H["LLM 回答"]
    H --> I["答案 / 引用 / 附件"]
```

你练习时要补充的问题：

- 哪一步开始受权限影响？
- 哪一步是结构化，哪一步是非结构化？
- 哪一步决定了回答是否可追溯？

## 6. 10 分钟讲解稿模板

### 开场

这套系统是一个以 Excel 为主数据源的 RAG 问答系统，面向政策、招标、企业三类业务数据。

### 第一部分：数据怎么进来

- 原始来源包括网页、PDF、扫描件等
- 当前项目先把采集结果整理成 Excel，再通过导入接口进入系统
- 导入时会做字段映射、校验、入库

### 第二部分：为什么能搜到

- 结构化字段支持精确过滤
- 长文本字段会切块并做向量化
- 查询时会把结构化检索和语义召回合并

### 第三部分：为什么能回答

- 问题先进入检索
- 检索结果作为证据传给 LLM
- 系统返回答案时会附带引用和附件

### 第四部分：系统为什么可信

- 有权限过滤
- 有引用来源
- 证据不足时会保守回答

### 结尾

如果继续扩展，这套系统可以往网页采集、OCR、BM25、评测体系和召回优化方向演进。
